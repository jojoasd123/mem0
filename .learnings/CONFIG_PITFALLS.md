# 配置修改经验

## 核心原则

> **修改自身配置时，先查官方文档，再看别人的解决方案。**

## 原因

1. **官方文档有配置示例** —— 虽然难读，但最准确
2. **避免踩坑** —— 别人的方案可能过时或不适用
3. **OpenClaw 特性** —— 官方文档在 https://docs.openclaw.ai/

## 实际踩坑

### 2026-02-05 升级后 Gateway 无法访问
**原因**: 直接修改 `allowInsecureAuth: false` 没查文档
**后果**: Web UI 无法访问，需要回退
**教训**: 应该先查文档了解该配置的影响范围

## 工作流程

修改配置前：
1. 🔍 打开 https://docs.openclaw.ai/
2. 📖 搜索相关配置项
3. ✅ 确认示例和副作用
4. ⚠️ 不确定时先问哥哥
5. 🛠️ 再执行修改

## 文档特点

- 作者：AI 生成
- 风格：几乎没有重点，不人性化
- 价值：配置示例准确、全面
- 技巧：用搜索定位，不要通读

# 任务拆解器 - OpenClaw Skill

自动将复杂任务拆解为可并行执行的子任务。

## 触发方式

**自然语言触发**：
- "拆解任务：创建一个待办事项应用"
- "并行执行：修复 bug 并写文档"
- "把这个任务分解一下：xxx"
- "多任务处理：xxx"

## 工作原理

1. **LLM 拆解**：用 qwen3.5-plus 分析任务，拆解为独立子任务
2. **Agent 分配**：根据子任务类型分配给 main/coder/video
3. **并行执行**：同时启动多个子代理（sessions_spawn）
4. **结果汇总**：收集所有结果，整合回复

## 示例

**输入**：
```
拆解任务：创建一个待办事项应用，包括前端、后端和数据库
```

**输出**：
```
🔍 任务拆解完成（任务复杂，适合并行执行）

📋 拆解为 3 个并行子任务：

1. **CODER**: 设计数据库 schema，创建用户和任务表
2. **CODER**: 实现后端 API（用户认证、CRUD 操作）
3. **CODER**: 创建前端界面（任务列表、添加/完成功能）

🚀 开始并行执行...

[等待子代理执行...]

📊 执行结果汇总

✅ 子任务 1: 设计数据库 schema...
   > 已创建 users 表和 tasks 表，包含...

✅ 子任务 2: 实现后端 API...
   > 已完成用户认证和 CRUD 接口...

✅ 子任务 3: 创建前端界面...
   > 已完成 React 组件和样式...

✨ 完成 3/3 个子任务
```

## 配置

在 `openclaw.json` 中启用：

```json
{
  "skills": {
    "entries": {
      "task-decomposer": {
        "enabled": true
      }
    }
  }
}
```

## 文件结构

```
skills/task-decomposer/
├── SKILL.md           # 技能说明（本文件）
├── __init__.py        # 技能入口
├── decompose.py       # 核心拆解逻辑
└── scripts/
    └── router.py      # 路由脚本（可选）
```

### 决策1：Claude Code 架构从本地中转改为云端SSH直连
**决策点：** 放弃"本地芋圆中转"方案，采用"云端芋圆直接SSH控制"

**原因：**

**新架构：**

**决策点：** 设计Claude Code完成后主动回调飞书通知

**原因：**
- 需要实时反馈任务执行结果
- 用户需要知道任务状态（进行中/完成/失败）
- 避免频繁轮询造成的资源浪费

#### 2. SSH 密钥管理核心原则

##### 命名规范
- **后缀区分类型**:
  - `.key` / 无后缀 → SSH 私钥
  - `.pem` → 可能是 SSH 私钥或 SSL 证书
  - `.pub` → 公钥

- **用途前缀**:
  - `yuyuan12.top.*` → 域名相关
  - `gitea*` → Gitea 相关
  - `wsl2*` → WSL2 相关

##### 权限管理
```bash
# SSH 私钥必须 600
chmod 600 ~/.ssh/private_key.pem

# 公钥可以 644
chmod 644 ~/.ssh/public_key.pub
```

##### 密钥格式识别
```bash
# 查看密钥类型
file ~/.ssh/secret_key

# 输出示例：
# OpenSSH private key → SSH 私钥
# PEM RSA private key → SSH 私钥（传统格式）
# OpenPGP public key → GPG 公钥
# PEM certificate → SSL 证书
```

#### 3. 连接测试流程

##### Step 1: 确认密钥类型
```bash
file ~/.ssh/some_key
```

##### Step 2: 检查权限
```bash
ls -la ~/.ssh/some_key
# 应该是 -rw------- (600)
```

##### Step 3: 尝试连接（带调试）
```bash
ssh -i ~/.ssh/some_key -v user@host
# 查看日志中的 "Offering public key" 确认密钥是否被使用
```

##### Step 4: 排查常见错误

| 错误信息 | 原因 | 解决方法 |
|---------|------|---------|
| `Permission denied (publickey)` | 密钥不匹配 | 检查 authorized_keys |
| `Load key ... error in libcrypto` | 密钥格式错误 | 可能是 SSL 证书 |
| `Permissions 0644` | 权限太开放 | chmod 600 |
| `Could not resolve hostname` | 主机名错误 | 检查 DNS/hosts |

#### 6. 新项目 SSH 密钥管理流程

##### 初始化阶段
1. **生成密钥**:
   ```bash
   ssh-keygen -t ed25519 -C "project_name@user" -f ~/.ssh/project_name
   ```

2. **设置权限**:
   ```bash
   chmod 600 ~/.ssh/project_name
   chmod 644 ~/.ssh/project_name.pub
   ```

3. **添加到 Config**:
   ```bash
   Host project-name
       HostName server.com
       User username
       IdentityFile ~/.ssh/project_name
   ```

4. **更新文档**:
   ```markdown
   - 在 KEYS.md 添加记录
   - 在项目文档中说明连接方式
   ```

##### 部署阶段
1. **上传公钥到服务器**:
   ```bash
   ssh-copy-id -i ~/.ssh/project_name.pub user@server
   ```

2. **测试连接**:
   ```bash
   ssh project-name
   ```

3. **记录授权密钥**:
   ```bash
   cat ~/.ssh/authorized_keys > /path/to/project/docs/authorized_keys.txt
   ```

##### 维护阶段
- **定期审查**: 删除不再使用的密钥
- **权限检查**: `find ~/.ssh -type f -name "*.pem" -exec chmod 600 {} \;`
- **备份加密**: 使用 GPG 加密备份私钥

#### 7. 安全注意事项

##### ❌ 绝对禁止

- 在对话中输出私钥内容
- 共享私钥给他人
- 在公网服务器上使用弱密码密钥

- 使用 ed25519 算法（更安全、更小）

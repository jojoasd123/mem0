### 今天的踩坑全记录

#### 坑点 1：端口混淆
**错误**：连接 Windows 时用了 WSL 的端口 2222
**正确**：Windows SSH 默认端口是 **22**
**教训**：WSL 和 Windows 的端口是分开的！

#### 坑点 2：authorized_keys 文件名没辨识度
**问题**：Windows 上已有多个 SSH 密钥，authorized_keys 太乱
**解决方案**：修改 `C:\ProgramData\ssh\sshd_config`，添加自定义密钥文件：
```
AuthorizedKeysFile .ssh/yuyuan_cloud.pub
```

#### 坑点 3：Windows OpenSSH 权限要求极其严格
**表现**：文件内容正确，但公钥认证仍失败
**教训**：Windows OpenSSH 对文件权限的要求比 Linux 更严格！
**必须执行的权限修复命令**：
```powershell
$sshDir = "$env:USERPROFILE\.ssh"
$keyFile = "$sshDir\yuyuan_cloud.pub"

# 禁用继承
icacls $keyFile /inheritance:r

# 只允许当前用户和 SYSTEM
icacls $keyFile /grant:r "$($env:USERNAME):(R)"
icacls $keyFile /grant:r "SYSTEM:(R)"

# 重启服务
Restart-Service sshd -Force
```

#### 坑点 4：文件编码问题
**错误**：PowerShell 的 `Out-File` 默认是 UTF-16
**正确**：必须用 `-Encoding ASCII` 参数
**教训**：Windows OpenSSH 只认 ASCII/UTF-8 编码的密钥文件！

#### 坑点 5：重复添加公钥
**问题**：authorized_keys/yuyuan_cloud.pub 里重复了多条相同公钥
**解决**：用 `-Force` 覆盖，不要用 `-Append` 追加

### 下次配置的完整步骤（备忘）

```powershell
# === 完整步骤：Windows SSH 免密配置 ===

# 1. 安装并启动 OpenSSH Server
Add-WindowsCapability -Online -Name OpenSSH.Server~~~~0.0.1.0
Start-Service sshd
Set-Service -Name sshd -StartupType Automatic

# 2. 允许防火墙
New-NetFirewallRule -Name "OpenSSH-Server" -DisplayName "OpenSSH Server" -Enabled True -Direction Inbound -Protocol TCP -Action Allow -LocalPort 22

# 3. 修改 sshd_config，添加自定义密钥文件
Add-Content -Path "C:\ProgramData\ssh\sshd_config" -Value "`nAuthorizedKeysFile .ssh/yuyuan_cloud.pub`n"

# 4. 创建 .ssh 目录（如果不存在）
$sshDir = "$env:USERPROFILE\.ssh"
if (!(Test-Path $sshDir)) { New-Item -ItemType Directory -Force -Path $sshDir }

# 5. 写入公钥（ASCII编码！）
$pubKey = "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIJphnPwnzXwwWmHLiPcrzdHdHNmLJGp3g+Gm292Ge9at yuyuan-cloud\(WSL\)"
$pubKey | Out-File -FilePath "$sshDir\yuyuan_cloud.pub" -Encoding ASCII -Force

# 6. 修复权限（极其重要！）
icacls "$sshDir\yuyuan_cloud.pub" /inheritance:r
icacls "$sshDir\yuyuan_cloud.pub" /grant:r "$($env:USERNAME):(R)"
icacls "$sshDir\yuyuan_cloud.pub" /grant:r "SYSTEM:(R)"

# 7. 重启 SSH 服务
Restart-Service sshd -Force

# 8. 本地测试（验证免密）
ssh -o BatchMode=yes neko@localhost "echo 测试成功"
```

### git-commit-squash-weekly 任务改进
- **原因**: 当初设置任务时只有一个仓库，现在有 yuyuan12、yuyuan-skills、yuyuan-memory-sync 三个仓库
- **旧策略**: 提交数>5就全部 squash，会误伤功能提交
- **新策略（方案A）**: 智能识别 "Backup sync:" 类型，仅合并自动备份提交
- **覆盖仓库**: 三个仓库都会检查，按需处理
- **触发条件**: 备份类提交 >= 3 才 squash
- **下次运行**: 明天（周日）凌晨 04:00

## 学到的经验

1. Gateway 的 `update.run` 会自动重启服务，但 PID 可能不变，需要用 `openclaw gateway restart` 确保加载新版本
2. npm 安装新版本时耗时较长（约1分多钟），需要耐心等待
3. 升级前提交 Git 改动是好习惯
4. **改进任务/配置时，要检查影响范围** —— 比如 git-commit-squash-weekly 当初的设置已经不适应现在的多仓库结构

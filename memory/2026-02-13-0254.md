## 为什么需要重启？

因为我直接修改了 `jobs.json` 文件，而不是通过 CLI 工具。**Gateway 启动时会将 cron 配置加载到内存**，之后运行时不再监控文件变化。

- **`~/.openclaw/cron/jobs.json`** 是持久化存储
- **Gateway 内存中的 cron 配置** 是运行时使用的

当我修改文件后，Gateway 不知道文件变了，还在用旧的内存配置。所以需要重启让 Gateway 重新读取文件。

## 应该怎么做？

```bash
# 修改单个任务
openclaw cron edit 08491a9d-76a6-44a2-89bd-947fbcfa6de2 --cron "5 */6 * * *"

# 这样 Gateway 会自动热加载新配置，不需要重启！
```

**教训**：修改 cron 任务应该用 `openclaw cron edit`，而不是直接改 `jobs.json` 文件。CLI 工具会通过 WebSocket 通知 Gateway 更新配置，支持热加载！🎯

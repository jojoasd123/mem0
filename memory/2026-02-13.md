## 2. 本地芋圆 WSL2 故障排查

### 问题诊断
- **症状**: 本地芋圆无法启动
- **根因**: WSL2 的 systemd 未正确启用

### 关键错误信息
```
systemctl --user unavailable: Failed to connect to bus: No such file or directory
wsl: Unknown key 'network.systemd' in /etc/wsl.conf:2
```

### 修复步骤
1. **修正 `/etc/wsl.conf`**（删除错误的 `network.systemd` 行）
   ```bash
   echo "[boot]
   systemd=true" | sudo tee /etc/wsl.conf
   ```

2. **在PowerShell重启WSL**
   ```powershell
   wsl --shutdown
   wsl -d Ubuntu-24.04
   ```

3. **验证systemd**
   ```bash
   systemctl is-system-running  # 应显示 running
   ```

4. **启动服务**
   ```bash
   sudo systemctl start tailscaled
   sudo tailscale up
   openclaw gateway start
   ```

### 环境信息
- **WSL发行版**: Ubuntu-24.04
- **版本**: 2
- **已切换**: 从社区版 `openclaw-cn` 切回官方版 `openclaw`

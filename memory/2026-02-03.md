**OpenClaw 的配对码 / Token 并不是在日志中生成的**，而是：

## 1. Gateway Token（固定令牌）
存储在配置文件里：
```bash
~/.openclaw/openclaw.json → gateway.auth.token
```
这个 Token 是首次 `onboard` 或 `doctor --generate-gateway-token` 时生成的，**不会随每次启动改变**。

## 2. 设备配对码（临时）
当使用 CLI 配对时（如 `openclaw pairing` 或连接 WebSocket），系统会生成**临时的配对请求码**，比如 `JN4MSY23`。

从日志里看到的：
```
device pairing auto-approved device=... role=operator
```
意思是：**设备被自动批准了**，不需要手动输入配对码。这是因为在 `USER.md` 里你已经被标记为 operator。

## 3. Telegram 配对
```
telegram pairing request  → 用户首次发消息时会触发配对请求
```

---

- 配对码不是"生成在日志中"供你查看的
